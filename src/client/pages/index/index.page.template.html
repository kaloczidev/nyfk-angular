Az inputokbol valo adatot a hozza tartozo komponens osztalyahoz kotve real-time tudjuk a valtozasokat kovetni.
pl: PersonComponent data mezoje. lathato, hogy a templateban az inputokon egy [(ngModel)]="data.name" szeru dolog van, ami az adott input erteket koti a megadott mezohoz. Ehhez a FormsModule szukseges (lasd modulhasznalat, pl: HttpModule).
Ez a ketiranyu adatkotes lenyege, valtozaskor az osztaly mezojenek erteke valos idoben valtozik.
Routolas: App.module-ban lathato pelda, egy komponenst kell megadni amit a router majd cserelget az adott url-nek megfeleloen.
useHash: # alkalmazasa az urlben, hogy veletlenul se kuldjon a szervernek egy masik kerest. ( # ugye oldalon beluli navigaciora szolgal).
layout: egy wrapper az egesz tartalomnak, ez tartalmazza a fejlecet es a lablecet. a benne levo tartlom szamara helyet biztosit, es ezen belul cserelgeti a router a tartalmat. Erre a celra vannak a PageComponent-ek.

Ez a repo egy kiindulasi alap, ezt ajanlott szemelyre (feladatra) szabni.
